## Installation

```bash
cargo add swarms-rs
```

<Note>
  Requires **Rust 1.70+**. Update your toolchain with `rustup update` if needed.
</Note>

## Environment variables

```bash
OPENAI_API_KEY="your_openai_api_key"
DEEPSEEK_API_KEY="your_deepseek_api_key"
```

## Quick start

```rust
use std::env;
use anyhow::Result;
use swarms_rs::{llm::provider::openai::OpenAI, structs::agent::Agent};

#[tokio::main]
async fn main() -> Result<()> {
    dotenv::dotenv().ok();

    let api_key = env::var("OPENAI_API_KEY").expect("OPENAI_API_KEY must be set");
    let client = OpenAI::new(api_key).set_model("gpt-4-turbo");

    let agent = client
        .agent_builder()
        .system_prompt("You are a helpful assistant.")
        .agent_name("BasicAgent")
        .user_name("User")
        .build();

    let response = agent
        .run("Tell me about Rust programming.".to_owned())
        .await?;

    println!("{}", response);
    Ok(())
}
```

## Core configuration

| Parameter | Description | Required |
| --- | --- | --- |
| `system_prompt` | Role and instructions for the agent | Yes |
| `agent_name` | Agent identifier | Yes |
| `user_name` | User identifier in the session | Yes |
| `max_loops` | Max reasoning loops per run | No (default `1`) |
| `retry_attempts` | Retries on failure | No |
| `enable_autosave` | Persist state between runs | No |
| `save_state_dir` | Directory for saved state | No |

<Tip>
  Start with low `max_loops` and increase only when tasks need deeper multi-step reasoning — each loop may add API cost.
</Tip>

## MCP tools

Agents can attach MCP servers over STDIO or SSE:

```rust
let agent = client
    .agent_builder()
    .system_prompt("You are a helpful assistant with access to tools.")
    .agent_name("ToolAgent")
    .user_name("User")
    .add_stdio_mcp_server("uvx", ["mcp-hn"])
    .await
    .add_sse_mcp_server("file-browser", "http://127.0.0.1:8000/mcp")
    .await
    .build();
```

### Setup helpers

```bash
# uv (package runner)
curl -sSf https://astral.sh/uv/install.sh | sh

# Example STDIO server
uvx install mcp-hn

# Example SSE filesystem proxy
uvx mcp-proxy --sse-port=8000 -- npx -y @modelcontextprotocol/server-filesystem ~
```

<Warning>
  Filesystem and shell MCP tools can access sensitive paths. Restrict scope in production.
</Warning>

## Specialized agent example

```rust
let agent = client
    .agent_builder()
    .system_prompt("You are a cryptocurrency analysis assistant...")
    .agent_name("CryptoAnalyst")
    .user_name("Trader")
    .enable_autosave()
    .max_loops(3)
    .save_state_dir("./crypto_analysis/")
    .build();

let response = agent
    .run("What are your thoughts on Bitcoin's current market position?".to_owned())
    .await?;
```

## FAQ

<AccordionGroup>
  <Accordion title="Which LLM providers are supported?">
    OpenAI and DeepSeek are supported today; additional providers are planned.
  </Accordion>
  <Accordion title="What is the difference between max_loops and retry_attempts?">
    `max_loops` limits reasoning steps per query. `retry_attempts` limits how many times a failed call is retried.
  </Accordion>
  <Accordion title="Can I use tools without MCP?">
    Tooling is designed around MCP for a standard interface to external systems.
  </Accordion>
</AccordionGroup>
