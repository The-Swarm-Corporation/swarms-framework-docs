---
title: Group Chat
description: An asynchronous, self-selecting multi-agent conversation where each agent decides for itself whether to reply
---

`GroupChat` runs an asynchronous conversation where every agent gets its own coroutine. Each agent independently evaluates incoming messages, scores its own desire to respond, and only broadcasts a reply when that score clears a configurable `threshold`. There are no rounds and no speaker-selection function — any agent (or several at once) can chime in whenever it has something useful to add.

## When to Use

- **Debate and discussion**: Multiple perspectives on complex topics
- **Collaborative problem-solving**: Agents work together through open conversation
- **Brainstorming**: Generate ideas through multi-agent interaction
- **Open-ended panels**: Let participants speak only when they have something to add, instead of forcing every agent to respond every round

## Key Features

- Asynchronous, self-selecting participation — no fixed speaking order
- Each agent scores its own desire to respond (`0..1`) via a forced `respond` tool call
- Configurable `threshold` for how selective agents are
- Conversation ends after `idle_timeout` seconds of inactivity or `max_loops` total messages
- `auto_equip` automatically injects the `respond` tool into every agent
- Mixed model providers — each agent runs in its own coroutine, so participants can use different LLM providers concurrently
- Conversation history tracking via `Conversation`
- Inherits `SerializableMixin` — `to_dict()` is available for telemetry / persistence

## Basic Example

```python
from swarms import Agent, GroupChat

tech_optimist = Agent(
    agent_name="TechOptimist",
    system_prompt="You argue for the benefits of AI in society.",
    model_name="gpt-5.4",
    max_loops=1,
    persistent_memory=False,
)

tech_critic = Agent(
    agent_name="TechCritic",
    system_prompt="You argue against unchecked AI advancement.",
    model_name="gpt-5.4",
    max_loops=1,
    persistent_memory=False,
)

realist = Agent(
    agent_name="Realist",
    system_prompt="You weigh both sides and seek a balanced view.",
    model_name="gpt-5.4",
    max_loops=1,
    persistent_memory=False,
)

chat = GroupChat(
    name="AI-Ethics-Debate",
    description="Discussion on AI's societal impact",
    agents=[tech_optimist, tech_critic, realist],
    max_loops=10,     # hard cap on total posted messages
    threshold=0.5,    # minimum self-rated score required to speak
)

result = chat.run("Should we prioritize AI development or AI regulation?")
print(result)
```

<Note>
  `GroupChat` requires at least **two** agents — it raises `ValueError` otherwise.
</Note>

## How Agents Decide to Speak

There is no speaker-selection function. Instead, each agent's coroutine:

1. Waits for a message in its inbox (the initial task, or a broadcast from another agent).
2. Snapshots the current conversation history.
3. Is asked — via a forced `respond` tool call — to return a `(score, message)` pair: `score` is `0..1` for how much it wants to reply, and `message` is its reply (or an empty string).
4. Broadcasts `message` to every other agent's inbox **only if** `score > threshold`.

Multiple agents can score above threshold for the same message and reply concurrently — there's no turn-taking.

The conversation stops when either:

- `max_loops` total messages have been posted (the initial task counts as message 1), or
- `idle_timeout` seconds pass with no new messages.

### The `respond` tool

`auto_equip=True` (the default) appends `RESPOND_TOOL` to each agent's `tools_list_dictionary` and rebuilds its LLM client so every model call is forced to return a structured score/message decision:

```python
from swarms.structs.groupchat import RESPOND_TOOL

print(RESPOND_TOOL)
# {
#     "type": "function",
#     "function": {
#         "name": "respond",
#         "description": "Decide whether to reply in the groupchat. Set score 0..1 ...",
#         "parameters": {
#             "type": "object",
#             "properties": {
#                 "score": {"type": "number", "minimum": 0, "maximum": 1},
#                 "message": {"type": "string"},
#             },
#             "required": ["score", "message"],
#         },
#     },
# }
```

If you set `auto_equip=False`, your agents must already carry a compatible `respond` tool — otherwise their speaking decisions can't be parsed and they'll stay silent for the whole chat.

## Key Parameters

<ParamField path="name" type="str" default='"dynamic-groupchat"'>
  Human-readable name used in logs and serialized state.
</ParamField>

<ParamField path="description" type="str" default='"Agents choose whether to speak at any time."'>
  Short description of the chat's purpose.
</ParamField>

<ParamField path="agents" type="Optional[List[Agent]]" required>
  Participating agents. At least 2 are required.
</ParamField>

<ParamField path="max_loops" type="int" default="20">
  Maximum number of messages posted before stopping, including the initial task.
</ParamField>

<ParamField path="threshold" type="float" default="0.5">
  Minimum self-rated score (`0..1`) required for an agent's reply to be broadcast. Raise it to make agents more selective; lower it for livelier chats.
</ParamField>

<ParamField path="idle_timeout" type="float" default="8.0">
  Seconds of inactivity after which the conversation ends.
</ParamField>

<ParamField path="output_type" type="str" default='"str-all-except-first"'>
  Format passed to `history_output_formatter` for the returned conversation history. See [Output Formats](#output-formats).
</ParamField>

<ParamField path="verbose" type="bool" default="False">
  Emit internal log messages (e.g. injected tools, broadcast activity).
</ParamField>

<ParamField path="print_on" type="bool" default="True">
  Print each broadcast message to the terminal as it happens, via `formatter.print_panel`.
</ParamField>

<ParamField path="auto_equip" type="bool" default="True">
  Automatically inject `RESPOND_TOOL` into every agent's `tools_list_dictionary` and rebuild its LLM client.
</ParamField>

## Methods

### `run(task)`

Synchronously run the groupchat until it idles out or hits `max_loops`. Internally wraps the async implementation in `asyncio.run()`.

```python
result = chat.run("Discuss the future of quantum computing")
```

<Note>
  Because `run()` calls `asyncio.run()`, it cannot be called from inside an already-running event loop (notebooks, async frameworks). In those contexts, call the internal async implementation directly or run the chat in a separate thread.
</Note>

### `run_batch(tasks)`

Run multiple tasks in sequence, one complete conversation per task.

```python
tasks = [
    "Analyze tech sector trends",
    "Evaluate real estate market",
    "Review commodity prices",
]

results = chat.run_batch(tasks)
```

### `to_dict()`

Inherited from `SerializableMixin`. Returns a JSON-friendly snapshot of the chat configuration (excludes `agents` and `conversation`).

## Use Cases

### Debate

```python
pro_agent = Agent(agent_name="ProAI", system_prompt="Argue for AI benefits", model_name="gpt-5.4")
con_agent = Agent(agent_name="ConAI", system_prompt="Argue AI risks", model_name="gpt-5.4")
judge = Agent(agent_name="Judge", system_prompt="Evaluate arguments", model_name="gpt-5.4")

debate = GroupChat(
    name="AI-Debate",
    agents=[pro_agent, con_agent, judge],
    max_loops=12,
    threshold=0.6,  # judge and debaters should be selective
)
verdict = debate.run("Should AI development be regulated?")
```

### Expert Panel

```python
literature_expert = Agent(agent_name="LitExpert", model_name="gpt-5.4")
data_scientist = Agent(agent_name="DataSci", model_name="gpt-5.4")
statistician = Agent(agent_name="Stats", model_name="gpt-5.4")
writer = Agent(agent_name="Writer", model_name="gpt-5.4")

research_chat = GroupChat(
    name="Research-Collaboration",
    agents=[literature_expert, data_scientist, statistician, writer],
    max_loops=20,
    idle_timeout=15.0,  # give agents more time to think between contributions
)

paper = research_chat.run("Collaborate on a research paper about machine learning")
```

### Customer Service Team

```python
technical_support = Agent(agent_name="TechSupport", model_name="gpt-5.4")
billing_specialist = Agent(agent_name="Billing", model_name="gpt-5.4")
product_expert = Agent(agent_name="ProductExpert", model_name="gpt-5.4")
escalation_manager = Agent(agent_name="Manager", model_name="gpt-5.4")

support_team = GroupChat(
    name="Support-Team",
    agents=[technical_support, billing_specialist, product_expert, escalation_manager],
    max_loops=15,
)

result = support_team.run("Customer reports their account was charged twice this month.")
```

## Output Formats

`output_type` is passed straight to `history_output_formatter`. Common values:

```python
# Plain string, excluding the first (task) message — the default
chat = GroupChat(agents=agents, output_type="str-all-except-first")

# Full conversation as a dict
chat = GroupChat(agents=agents, output_type="dict")

# Full conversation as a list
chat = GroupChat(agents=agents, output_type="list")

# Only the final message
chat = GroupChat(agents=agents, output_type="final")
```

Other accepted values include `"string"`, `"dict-all-except-first"`, `"json"`, `"yaml"`, `"xml"`, and `"all"` — see `swarms.utils.output_types.HistoryOutputType` for the full list.

## Best Practices

<Note>
  **Threshold tuning**: a higher `threshold` (e.g. `0.6`–`0.8`) produces tighter, more selective conversations; a lower `threshold` (e.g. `0.3`) produces livelier chats where more agents chime in on each message.
</Note>

1. **Clear roles** — give agents distinct perspectives/expertise so the `respond` decision (based on relevance to their expertise) is meaningful.
2. **Set `idle_timeout` deliberately** — too short and slow models won't get a chance to reply; too long and a quiet chat will run until `max_loops`.
3. **Cap `max_loops`** — this is a hard ceiling on total messages (including the initial task), so set it based on how many exchanges you actually want.
4. **At least two agents** — `GroupChat` raises `ValueError` with fewer than 2.
5. **Leave `auto_equip=True`** unless your agents already define a compatible `respond` tool.
6. **Per-agent settings** — set `max_loops=1` on each agent so it makes one decision per inbox message, and `persistent_memory=False` so every decision is made fresh from the shared conversation history rather than the agent's own memory.

<Warning>
  Conversation length and cost scale with `max_loops` and the number of agents — every agent is asked to score every message that arrives in its inbox, even if it ultimately stays silent.
</Warning>

## When NOT to Use

- **Simple tasks** — use a single `Agent`.
- **Independent analysis** — when agents shouldn't influence each other, use [ConcurrentWorkflow](/architectures/concurrent-workflow).
- **Strict ordering** — when a fixed sequence is required, use [SequentialWorkflow](/architectures/sequential-workflow).
- **Director-led delegation** — use [HierarchicalSwarm](/architectures/hierarchical-swarm).

## Error Handling

```python
try:
    result = chat.run("Task")
except ValueError as e:
    print(f"Configuration error: {e}")
    # e.g. fewer than 2 agents
except Exception as e:
    print(f"Execution error: {e}")
    # e.g. runtime errors from individual agents
```
